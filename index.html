02 — projects

Selected work

betops

A real-time prediction market platform, built solo end-to-end across a hybrid cloud setup spanning AWS and a self-hosted Kubernetes cluster.

terraformawskubernetesgithub actionstailscale
// how it works

The backend is split into small, single-purpose services rather than one monolith — each one owns a distinct piece of the flow, from handling incoming requests to processing data asynchronously and persisting it. Cloud-native AWS services handle the parts that need to scale on demand, while the heavier compute runs on a self-hosted k3s cluster on Proxmox. The two environments talk to each other over a private, encrypted Tailscale connection rather than anything exposed to the public internet. Everything — infra and app — ships through one CI/CD pipeline, so a commit can go from code to a running deployment on either side without manual steps.

nebulay

A self-built homelab that doubles as a hybrid-cloud playground — production-style infrastructure running on hardware sitting in the same room as me.

proxmoxkubernetesawstailscalegrafana
// how it works

A Proxmox host runs a set of Linux VMs that form a k3s Kubernetes cluster, treated like a real production environment rather than a sandbox. That cluster connects out to private AWS subnets over a Tailscale VPN, so workloads on either side can reach each other securely without anything facing the public internet. The goal was to get hands-on with the same patterns used in real hybrid-cloud setups — provisioning, networking, scaling, and monitoring — on infrastructure I fully own and can break (and fix) without consequences.
